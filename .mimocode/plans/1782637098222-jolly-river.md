# Plan: Fix find_ip cycle data persistence and display (v2)

## Root Cause Analysis (from user's latest feedback)

Three bugs remain after v1:

### Bug 1: Zero upload/download values in JSON
When an IP is removed and re-added, the old connection's byte counters in `IpByteCounters` persist. The new IP inherits them. If the old proxy connection is destroyed between removal and snapshot recording, the counters drop to 0, creating a zero snapshot.

### Bug 2: Cycle count > 1 even with DROP_COUNT=MAX_IP
`pre_scanned` entries added via `add_ip` don't clear `cycle_history`. So a removed-and-re-added IP retains old history entries from its previous lifetime.

### Bug 3: Snapshot recorded for just-added IPs with stale counters
`record_cycle` is called for ALL active IPs, including replacements that inherited byte counters from old connections or have no connections yet.

## Implementation Plan

### Change 1: Clear dead IP data on removal (`proxy.rs`)

In `find_ip_cycle_manager`, after removing dead IPs from the pool, clear their byte counters and connection counts so replacement IPs start fresh:

```rust
// After remove_ip loop:
for ip in &dead_ips {
    // Clear byte counters so replacement IPs don't inherit stale data.
    if let Some(c) = cmc.byte_counters.upload.get(ip) { c.store(0, Ordering::Relaxed); }
    if let Some(c) = cmc.byte_counters.download.get(ip) { c.store(0, Ordering::Relaxed); }
    if let Some(c) = cmc.byte_counters.cycle_upload.get(ip) { c.store(0, Ordering::Relaxed); }
    if let Some(c) = cmc.byte_counters.cycle_download.get(ip) { c.store(0, Ordering::Relaxed); }
    if let Some(c) = cmc.byte_counters.connections.get(ip) { c.store(0, Ordering::Relaxed); }
    // Clear cycle history so re-added IPs start fresh.
    p.cycle_history_mut().remove(ip);
}
```

### Change 2: Record snapshots BEFORE replacement, only for IPs with traffic (`proxy.rs`)

Move snapshot recording to BEFORE the replacement step. Only record for IPs that have connections (meaning they actually had traffic this cycle):

```rust
// Record snapshots for IPs that had traffic this cycle (BEFORE replacement).
{
    let mut p = cmc.pool.write().unwrap();
    let active = p.active_ips().to_vec();
    for ip in &active {
        let (upload, download) = cmc.byte_counters.total_bytes(ip);
        let conns = cmc.byte_counters.connection_count(ip);
        if conns > 0 || upload + download > 0 {
            p.record_cycle(*ip, cycle_num, upload, download);
        }
    }
}

// NOW clear counters and do replacement...
for ip in &dead_ips {
    p.remove_ip(ip);
    // clear byte counters and cycle_history for dead_ips
}
// Replace from pre_scanned
```

### Change 3: Add `cycle_history_mut()` to IpPool (`proxy.rs`)

```rust
pub fn cycle_history_mut(&mut self) -> &mut HashMap<IpAddr, Vec<CycleSnapshot>> {
    &mut self.cycle_history
}
```
